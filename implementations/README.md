# Web3 Ideathon Methodology

```text
Web3 Ideathon Methodology
├── First pilot: Solana
├── Ethereum implementation: ProvaOnChain
└── Future ecosystem adaptations
```

The methodology is independent of any single blockchain. Each implementation must justify the chosen infrastructure on technical and educational grounds.

No blockchain should be used only because it is fashionable, familiar, or expected. The selected ecosystem must serve a concrete requirement such as verifiability, auditability, interoperability, public timestamping, or coordination between parties.

All implementations must respect privacy, security, transparency, and responsible educational use. That means keeping personal data offchain, documenting scope honestly, and separating proof from content.

## Implementation family

- [Solana pilot](solana-pilot/README.md)
- [Ethereum implementation](ethereum/README.md)

## Shared rules

- keep student identities offchain;
- store only what is necessary for verification;
- document what is planned, simulated, or completed;
- prefer public evidence over opaque claims;
- treat educational content as open and reusable when possible.
