# Solana Pilot

## Context

The Solana pilot was the first implementation associated with the Web3 Ideathon methodology. It served as the initial educational pilot in the repository history and produced documentation, reflection, and project evidence for student teams.

## Pedagogical goal

The goal was to validate how students could move from problem discovery to a demonstrable MVP while collaborating through GitHub and presenting technical decisions clearly.

## Technologies used

- Solana as the protocol context for the pilot;
- GitHub for documentation and evidence;
- Web3 concepts for educational exploration;
- team-based MVP development;
- presentation and reflection artifacts.

## What was validated

The Solana pilot validated the educational workflow, team collaboration, evidence-based assessment, MVP development process, and technical presentation. It did not define the methodology as Solana-specific.

## Limitations

- it does not imply that every future implementation must use Solana;
- it does not justify exposing personal data onchain;
- it does not remove the need for a technical decision in each new pilot;
- it does not replace later architecture, privacy, or security analysis.

## Lessons learned

- documentation is part of the product in educational Web3 projects;
- the protocol must fit the use case;
- privacy constraints must be explicit from the start;
- repositories need a clear separation between method, pilot, and implementation.

## Solana-specific elements

- the original pilot context;
- the first Web3 implementation history;
- references to the Solana ecosystem in the educational record.

## Reusable methodology elements

- eight-stage discovery-to-pitch workflow;
- evidence-based assessment;
- GitHub-centered documentation;
- MVP framing;
- privacy-first decision making;
- honest status reporting.

## Protocol identity note

The Solana pilot validated the methodology, but it did not define the methodology as Solana-only or Solana-dependent.
