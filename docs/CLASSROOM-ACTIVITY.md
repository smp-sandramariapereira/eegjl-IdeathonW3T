# Classroom Activity

This module maps the ProvaOnChain pilot to the eight Web3 Ideathon stages.

## 1. Discovery

- Activity: identify the problem of proving academic evidence.
- Decision: focus on verifiability and privacy.
- Evidence: problem statement and scope note.

## 2. Ideation

- Activity: consider alternative approaches.
- Decision: compare a blockchain proof with a conventional database.
- Evidence: decision note and rationale.

## 3. Solution

- Activity: define the flow for issuing and checking evidence.
- Decision: separate proof data from educational content.
- Evidence: flow description.

## 4. Technology

- Activity: justify the use of Ethereum.
- Decision: use Ethereum only when public verification is required.
- Evidence: technology note and architecture draft.

## 5. MVP

- Activity: define the smallest verifiable proof.
- Decision: limit the MVP to hash, project identifier, stage, URI, issuer, and timestamp.
- Evidence: MVP scope and requirements.

## 6. Architecture

- Activity: separate onchain and offchain components.
- Decision: keep all personal data offchain.
- Evidence: architecture diagram and privacy model.

## 7. Development

- Activity: implement and test the contract and scripts.
- Decision: keep the code minimal and readable.
- Evidence: contract, tests, and scripts.

## 8. Pitch

- Activity: demonstrate issuance and verification.
- Decision: present the pilot honestly, including limits.
- Evidence: sample issuance, verification output, and documentation.

Ethereum should not be used when a conventional database is sufficient, when information must be erasable, or when there is no meaningful public-verification requirement.
